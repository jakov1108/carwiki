⚡ Executive Summary — 60 seconds

  • Auto Wiki is a technically capable Croatian car encyclopaedia with a modern dark-UI aesthetic, hierarchical data structure (brand → model → generation → variant) and a user-submission pipeline.
  • The strongest areas are aesthetic design and user control; the weakest are help & documentation, search flexibility and language consistency.
  • No help page, inline tooltips, or guided onboarding exist — users must self-discover how the car submission workflow works.
  • Resolving the top 5 issues could meaningfully increase new-user retention and community contribution volume.

📋 Overview

🎯

Site Purpose

A community-driven car encyclopaedia for Croatian-speaking audiences. Users browse a structured database of cars organised by brand, model, generation and engine variant; compare up to three variants side-by-side; read automotive blog articles; and register to submit new entries for admin review.

👁️

First UX Impressions

The dark-by-default theme and blue gradient typography give an immediate modern, technical feel — appropriate for an enthusiast audience. Navigation is concise and well-labelled. The home-page hero search is functional but the layout density escalates quickly once users enter the car browsing flow. New visitors receive no onboarding cues.

⚠️

Key UX Risks at a Glance

No contextual help Weak search Mixed language (HR + EN) No breadcrumbs No inline form validation Missing loading skeletons Consistent visual design Clear error messages

🔬 Heuristic Analysis

📡
Heuristic 1
Visibility of System Status
65
Benchmark ≈ 72  Needs Improvement
Observed Issues
  • Loading state is plain text "Učitavam…" with no spinner or skeleton UI
  • 4-step car submission has no step indicator/progress bar
  • No spinner during form submission; users cannot tell if the request was received
  • Blog carousel has no slide-position dots
  • Compare panel shows count, but no "1 more to go" type guidance
Impact & Recommendations
  • Add skeleton loaders for car cards and images
  • Show a numbered stepper on the car submission flow (Step 2 of 4)
  • Disable submit button and show spinner on form send
  • Add dot indicators to blog carousel
"So what?" — Users who don't know if an action succeeded often repeat it, creating duplicate submissions and frustration that drives abandonment.
🗣️
Heuristic 2
Match: System & Real World
72
Benchmark ≈ 75  Needs Improvement
Observed Issues
  • Car category filter labels are in English (Compact, Sedan, SUV, Sports, Electric) while all other UI copy is Croatian
  • Technical spec abbreviations (TSI, PHEV, BEV, DSG) appear in data tables without inline explanations — glossary is only on About page
  • Admin-review concept not explained anywhere at the point of submission
Impact & Recommendations
  • Translate category labels: "Kompaktni", "Limuzina", "SUV/Crossover", "Sportski", "Električni"
  • Add a small tooltip (?) next to spec abbreviations linking to the glossary
  • Add a one-line note at submission: "Vaš unos prolazi kroz admin pregled (1–2 dana)"
"So what?" — Language inconsistency creates cognitive friction for non-technical Croatian users, reducing their confidence and time-on-site.
🕹️
Heuristic 3
User Control & Freedom
76
Benchmark ≈ 75  Needs Improvement
Observed Issues
  • Multi-step car submission has no "Back" button — browser back is the only escape
  • No undo or edit for submitted car proposals (once sent, cannot be recalled)
  • Logout icon (mobile & desktop) has no tooltip text — accidental taps are unrecoverable without re-login
Impact & Recommendations
  • Add a "← Natrag" (Back) button inside each step of the submission flow
  • Show a "pending proposals" dashboard so users can see and edit before admin approval
  • Add a confirmation dialog before logout ("Jeste li sigurni?")
"So what?" — Feeling trapped or unable to fix mistakes pushes users to abandon forms, costing the site community-contributed content.
🔗
Heuristic 4
Consistency & Standards
70
Benchmark ≈ 75  Needs Improvement
Observed Issues
  • Navigation links: some have icons (Compare, Add Car, Login), others don't — inconsistent icon usage pattern
  • Dark-mode form inputs on Cars page use bg-slate-800; Login form uses bg-slate-900 — slight visual discrepancy
  • No breadcrumb navigation anywhere — deep hierarchy (Brand › Model › Generation › Variant) has no wayfinding trail
  • Active/selected state styling differs between pages (blue button vs outlined)
Impact & Recommendations
  • Standardise either all-icon or icon-for-action-items pattern across nav
  • Implement breadcrumbs on detail pages following the data hierarchy
  • Create a shared form input component to unify styles
"So what?" — Inconsistency erodes trust; users spend mental effort noticing differences instead of using the product.
🛡️
Heuristic 5
Error Prevention
67
Benchmark ≈ 70  Needs Improvement
Observed Issues
  • No inline / real-time validation on registration form (password length, email format) — errors only show on submit
  • No password strength indicator during registration
  • Car submission form: no input constraints or format hints (e.g., "Enter year as YYYY")
  • Compare feature: max 3 cars enforced silently — no notification when limit is reached
Impact & Recommendations
  • Add onBlur inline validation for email and password fields
  • Add a password-strength bar to the registration form
  • Show placeholder/helper text for expected numeric formats in spec fields
  • Toast notification when compare limit (3) is reached
"So what?" — Forms that only show errors after submission have measurably higher abandonment rates — inline validation improves completion by up to 22%.
🧠
Heuristic 6
Recognition vs. Recall
75
Benchmark ≈ 78  Needs Improvement
Observed Issues
  • Theme toggle button has no text label — icon-only interaction not universally understood
  • Logout button icon-only on desktop nav — could be confused with Settings/Profile
  • Car comparison spec rows: colour-coded best-value cells work well, but no legend explaining the green highlight
  • Home page search dropdowns (Brand → Model → Generation) not pre-populated with commonly accessed items
Impact & Recommendations
  • Add aria-label AND visible tooltip for icon-only buttons (theme, logout)
  • Add a legend/note near compare table: "🟢 = Best value"
  • Show most popular brands in the Brand dropdown by default
"So what?" — Making interface elements self-describing reduces support requests and improves accessibility compliance.
⚡
Heuristic 7
Flexibility & Efficiency of Use
58
Benchmark ≈ 70  High Risk
Observed Issues
  • Search only queries brand name and model name — cannot search by spec (engine size, horsepower, fuel type, year)
  • Category filter is limited to 5 options (Compact, Sedan, SUV, Sports, Electric) — no multi-select
  • No keyboard shortcuts or power-user features
  • No "recently viewed" cars or saved favourites
  • Shareable compare URL only works via variantId param — no human-readable permalink
Impact & Recommendations
  • Add advanced search / filter panel: fuel type, power range, year range, drive type
  • Implement multi-select category filter
  • Persist recently compared/viewed in localStorage for returning users
  • Generate shareable compare links with model slugs
"So what?" — Enthusiast users (the core audience) expect powerful search; inadequate filtering forces them to rely on Google to find cars on your own site.

Heuristic 8
Aesthetic & Minimalist Design
80
Benchmark ≈ 72  Good UX ✓
Observed Issues
  • Minor: gradient text headers (from-blue-400 to-cyan-400) overused — appears on almost every page title
  • Loading placeholders are plain text instead of content-appropriate skeleton screens
  • Dark-mode default may alienate some casual users who expect light UI
Impact & Recommendations
  • Reserve gradient headings for truly primary H1s; use solid weight for secondary titles
  • Replace text "Učitavam…" with animated skeleton cards
  • Default to system preference (prefers-color-scheme) instead of forced dark mode
"So what?" — Visual polish is above par and builds brand credibility; small refinements here will elevate the perception from "beta" to "production-ready".
🔧
Heuristic 9
Error Recovery & Handling
68
Benchmark ≈ 72  Needs Improvement
Observed Issues
  • Login/Register errors: good Croatian-language messages — positive
  • API fetch errors: silently caught in try/catch blocks with empty catch {} — user sees nothing
  • 404 page exists but offers no navigation shortcuts to get back on track
  • Compare page: failed variant fetch fails silently with no user feedback
Impact & Recommendations
  • Replace silent catches with toast notifications: "Greška pri učitavanju. Pokušajte ponovo."
  • Add quick-links on the 404 page ("Vrati se na početnu", "Pretraži automobile")
  • Implement global error boundary with retry UI
"So what?" — Silent API failures are invisible bugs that damage trust without the team knowing; surfacing them also gives users a path forward instead of staring at a blank screen.
❓
Heuristic 10
Help & Documentation
55
Benchmark ≈ 65  High Risk
Observed Issues
  • About page has an FAQ and glossary — positive — but it is not linked from within workflows where it matters
  • No contextual tooltips on any data fields (e.g., what is "Varijanta"?)
  • No new-user onboarding or welcome tour
  • Car submission workflow has no inline instructions per step
  • Compare feature: no help text explaining how to initiate a comparison
Impact & Recommendations
  • Add contextual "?" icon links to relevant FAQ entries near complex fields
  • Add step-level instructions inside the submission form ("Odaberite marku automobila koji želite dodati")
  • Link to the glossary from all spec tables that contain abbreviations
  • Consider an optional first-time welcome modal for new registrations
"So what?" — New users who cannot understand the data model or submission process give up before contributing, directly limiting the growth of the car database.

👔 Stakeholder Summary

  • 🎯
    Search is a business-critical gap. The inability to filter by engine power, fuel type, or year means car enthusiasts — the most valuable users — cannot efficiently find what they need. Adding advanced filtering is the single highest-ROI improvement available.
  • 📝
    Community contributions are the site's growth engine — and they are at risk. New users who want to add a car hit an opaque 4-step form with no guidance. Better in-flow documentation will directly increase submission volume and database completeness.
  • ⚡
    Form errors are a silent conversion killer. Both the registration and the car submission forms only show errors after submission, creating frustration that leads to user drop-off. Inline validation is a low-effort, high-impact fix that will increase sign-up completion rates.
  • 🌍
    Language consistency builds local brand trust. Croatian car buyers will feel more at home — and share the site more — when every label speaks their language, including the currently English category tags (Compact, Sedan, SUV, Sports, Electric).
  • 🔍
    Breadcrumbs improve SEO and navigation simultaneously. With a four-level hierarchy (Brand › Model › Generation › Variant), users regularly get lost on deep pages. Breadcrumbs cost little to implement and pay back in organic search ranking through better structured-data signals.
  • 🎨
    The visual design is the strongest asset — protect it. The clean, dark-themed, card-based UI looks professional and on-brand. Continued investment in skeleton loading screens, responsive polish, and removing the "beta" feel will support premium brand positioning.
  • 📊
    Overall UX score of 70/100 indicates a solid foundation with clear, achievable improvements. Addressing the top 3 issues alone (search, help, form validation) could realistically push the score above 80 — the benchmark for "Good UX" — within a single development sprint.
